# vibe:learn — Self-Improvement Engine

Captures patterns, anti-patterns, and quality scores to improve the framework across projects.

## When to Run

After any phase completes, or when a notable pattern is discovered during work.

## Steps

### 1. Pattern Capture
- What approach worked well? Record the exact steps.
- What approach failed? Record why and the alternative.
- What was surprising? Unexpected behavior or interactions.
- What was the context? Project type, size, tech stack.

### 2. Anti-Pattern Recording
- Pattern that caused bugs or slowdowns
- Root cause analysis
- How to detect early in future
- How to fix if it happens again

### 3. Quality Scoring
For each completed phase, score 1-5:
- Test coverage adequacy
- Code quality
- Documentation clarity
- Process smoothness
- Tool effectiveness

### 4. Output
- Append to `.vibe/learnings/patterns.md`
- Each entry: date, phase, pattern description, outcome, tags
- Update quality scores over time to track improvement
