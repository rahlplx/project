# vibe:think — Product Strategy & Design Thinking

Forces clarity on problem, users, MVP scope, and success metrics before code.

## When to Run

When starting a new feature, product, or significant change. Never skip.

## Steps

### 1. Problem Definition
- What specific problem are we solving?
- Who experiences this problem?
- What happens if we don't solve it?
- Why now?

### 2. User Analysis
- Primary users and their goals
- User journey map (current state)
- Pain points in current workflow

### 3. Solution Sketch
- 3 possible approaches (at minimum)
- For each: effort estimate, impact estimate, risk level
- Recommended approach with rationale

### 4. MVP Definition
- Core must-have features (vertical slice)
- Nice-to-have (cut list)
- What explicitly is NOT in scope

### 5. Success Metrics
- Leading indicators (during build)
- Lagging indicators (post-ship)
- How we'll measure each

### 6. Output
Write a concise think document to `plans/think-<topic>.md` with sections:
- Problem
- Users
- Solution options (min 3)
- Recommendation
- MVP scope
- Success metrics

## Roles Active
- CEO (strategy, market fit)
- Designer (user experience, journey)
