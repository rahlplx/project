# Auto Curation Patterns

## Discovered in v2.0.0 auto pipeline

- YAML validation — entries are machine-readable; always validate with a script before commit
- Category targets — set minimum tool count per category (3+) as acceptance criteria
- Web search is the research tool — GitHub stars are the primary verification signal
- Auto pipeline works for curation — think/plan/break/build maps cleanly to tool research
- No PR needed for catalog-only changes — direct push is fine (no code risk)
